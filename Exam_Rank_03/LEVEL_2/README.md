# 🇬🇧 English

## Level 2 — Exam Rank 03

Level 2 is the second and final stage of the exam. This level focuses on **recursion** and **backtracking** algorithms.

### Questions

| Question | Topic | Difficulty | Description |
|:---------|:------|:----------:|:------------|
| [n_queens](n_queens/) | Backtracking | ⭐⭐⭐ | N-Queens problem — place N queens on a chessboard |
| [permutations](permutations/) | Recursion | ⭐⭐⭐ | Print all permutations of a string in alphabetical order |
| [powerset](powerset/) | Recursion, Subset Sum | ⭐⭐⭐ | Find all subsets that sum to N |
| [rip](rip/) | Backtracking, Pruning | ⭐⭐⭐⭐ | Balance parentheses by minimum removal |

### Required Fundamentals

- **Recursion**: A function calling itself — all Level 2 questions require recursion
- **Backtracking**: When entering an invalid path, go back and try another
- **Pruning**: Cutting unnecessary branches early for performance gain
- **Time Complexity**: These problems typically have $O(n!)$ or $O(2^n)$ complexity

# 🇫🇷 Français

## Level 2 — Exam Rank 03

Le Level 2 est la deuxième et dernière étape de l'examen. Ce niveau met l'accent sur les algorithmes de **récursion** et de **backtracking (retour en arrière)**.

### Questions

| Question | Sujet | Difficulté | Description |
|:---------|:------|:----------:|:------------|
| [n_queens](n_queens/) | Backtracking | ⭐⭐⭐ | Problème des N-Reines — placer N reines sur un échiquier |
| [permutations](permutations/) | Récursion | ⭐⭐⭐ | Afficher toutes les permutations d'une chaîne par ordre alphabétique |
| [powerset](powerset/) | Récursion, Subset Sum | ⭐⭐⭐ | Trouver tous les sous-ensembles dont la somme vaut N |
| [rip](rip/) | Backtracking, Élagage | ⭐⭐⭐⭐ | Équilibrer les parenthèses par suppression minimale |

### Connaissances Requises

- **Récursion** : Une fonction qui s'appelle elle-même — toutes les questions du Level 2 nécessitent la récursion
- **Backtracking (Retour en arrière)** : Revenir en arrière quand un chemin invalide est emprunté
- **Élagage (Pruning)** : Couper les branches inutiles tôt pour gagner en performance
- **Complexité temporelle** : Ces problèmes ont généralement une complexité $O(n!)$ ou $O(2^n)$
